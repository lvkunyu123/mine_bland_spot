# 自适应动态组包 + 改进LRU分级缓存算法

基于模糊控制预判式链路切换的露天矿盲区数据传输系统——手机端自适应传输算法实现。

## 项目内容

本项目实现了面向露天矿作业场景的车载终端自适应数据传输系统，核心包含以下模块：

### 核心模块

| 模块 | 文件 | 功能 |
|------|------|------|
| 数据生成模块 | `data_generator.py` | 模拟矿车产生JSON格式的报文数据，支持高优/常规优先级、多场景 |
| 自适应动态组包模块 | `adaptive_packet.py` | 根据网络质量动态调整组包大小、发送间隔、重传次数 |
| 改进LRU分级缓存模块 | `lru_cache.py` | 两级缓存架构，高优数据不淘汰，低优数据按Score淘汰 |
| 模糊控制链路切换引擎 | `fuzzy_link_switch_core.py` | 基于RSRP/RSSI/丢包率的模糊推理，判断链路质量并决策切换 |
| 网络控制器 | `network_controller.py` | 常驻运行核心，协调各模块，处理数据发送、缓存、补发 |
| TCP传输模块 | `tcp_sender.py` | 与PC端通信，发送数据并接收ACK |
| PC端接收服务 | `dispatch_server.py` | PC端接收数据并存入SQLite数据库，支持缺号检测和补发指令 |
| 主程序入口 | `main_simulator.py` | 仿真测试入口，模拟网络波动和数据生成 |

### 数据格式

```json
{
    "vehicle_id": "A",
    "timestamp": 1717912345.678,
    "seq": 100,
    "priority": 1,
    "scene": "深坑",
    "data": {
        "alarm_type": "发动机过热",
        "temperature": 120
    }
}
```

| 字段 | 说明 |
|------|------|
| `vehicle_id` | 车牌号 |
| `timestamp` | 时间戳 |
| `seq` | 序列号 |
| `priority` | 优先级（1=高优，0=常规） |
| `scene` | 场景（网络质量良好/WiFi链路可用/蜂窝链路可用/无信号/深坑/开阔区/弱网边坡） |
| `data` | 传输数据（高优为告警信息，常规为GPS等） |

### 工作流程

```
随机生成数据
    ↓
网络质量检测（模糊控制引擎）
    ├─ 网络优良 → 动态组包 → 直接传输 → 成功→结束 / 失败→L1缓存
    ├─ 单链路可用 → 切换链路 → 动态组包传输 → 成功→结束 / 失败→L1缓存
    └─ 双链路失效 → 直接写入L1缓存
            ↓
    后台检测网络恢复
            ↓
    自动补发L1缓存数据
            ├─ 成功 → 从缓存删除
            └─ 失败 → 移入L2缓存
                    ↓
            定期补发L2缓存数据
```

## 项目的实用性

### 适用场景

- **露天矿山车辆数据传输**：矿车在深坑、边坡等弱网区域的数据可靠传输
- **移动设备多链路切换**：WiFi与蜂窝网络之间的智能切换
- **高可靠数据采集**：告警类高优先级数据的不丢包保障

### 核心特性

1. **自适应动态组包**
   - 组包大小：256B ~ 4096B，通过sigmoid函数平滑映射
   - 发送间隔：50ms ~ 2000ms，与网络质量负相关
   - 重传次数：1 ~ 3次，高优数据固定3次

2. **改进LRU分级缓存**
   - 两级缓存架构（L1主缓存 + L2补发缓存）
   - 高优数据绝对不淘汰
   - 低优数据按Score淘汰（年龄因子+空闲时间+访问频率）

3. **模糊控制链路切换**
   - 基于RSRP、RSSI、丢包率三输入的模糊推理
   - 五种网络状态：优秀/良好/预警/危险/失效
   - 预判式切换 + 保护期防乒乓

4. **动态重传机制**
   - 高优数据强制重传3次
   - 常规数据根据网络质量动态计算
   - 发送过程中网络变差立即停止并缓存

5. **PC端数据持久化**
   - SQLite数据库存储
   - 自动去重（基于vehicle_id+seq唯一约束）
   - 缺号检测与补发指令

## 用户如何开始项目

### 环境要求

- Python 3.7+
- 依赖库：`numpy`, `scikit-fuzzy`, `statsmodels`

### 安装依赖

```bash
pip install numpy scikit-fuzzy statsmodels
```

### 运行方式

#### 方式一：完整仿真测试（PC端+手机端都在本机）

1. **启动PC端接收服务**（终端1）
   ```bash
   python dispatch_server.py
   ```

2. **启动手机端仿真**（终端2）
   ```bash
   python main_simulator.py test 100
   ```
   - `test 100`：测试模式，运行100个周期
   - `forever`：常驻运行模式

#### 方式二：仅运行手机端逻辑

```bash
python main_simulator.py forever
```

### 查看数据

运行后PC端会生成 `mine_data.db` 数据库文件，可使用SQLite工具查看。



### 辅助工具

| 工具 | 文件 | 用途 |
|------|------|------|
| 数据库读取工具 | `read_db.py` | 查看 `mine_data.db` 中的数据 |
| 数据去重工具 | `clean_duplicate.py` | 清理数据库中的重复记录 |

使用方式：
```bash
# 查看数据库
python read_db.py --stats
python read_db.py output.json

# 清理重复数据
python clean_duplicate.py
```
